# Модуль «Deskew» — устранение перекоса документа

### 1. Назначение
Обнаружить на сканированном листе документ, определить его границы, вычислить угол перекоса (в диапазоне примерно от -45° до +45°) и выровнять изображение так, чтобы строки текста стали горизонтальны, а края документа — параллельны рамке изображения. Модуль не определяет читаемую ориентацию текста (верх/низ) и не доворачивает на 90°, 180°, 270° — это задача следующего этапа.

### 2. Входные данные
- `imageBuffer` — буфер изображения в формате PNG или JPEG, **8-бит grayscale**, без альфа-канала.
- `options` (необязательный объект):
  - `edgeThreshold` — порог бинаризации градиента (0–255), по умолчанию `25`.
  - `dilateIterations` — число итераций дилатации (по умолчанию `2`).
  - `erodeIterations` — число итераций эрозии (по умолчанию `2`).
  - `padding` — отступ в пикселях после обрезки (по умолчанию `10`).

### 3. Выходные данные
Promise, разрешающийся объектом:
```typescript
{
  angle: number;           // угол перекоса в градусах, положительное направление — по часовой стрелке
  orientation: "portrait" | "landscape"; // соотношение сторон ограничивающего прямоугольника
  deskewedImage: Buffer;   // выровненное grayscale PNG-изображение (повёрнуто, обрезано, с паддингом)
}
```
Если документ не обнаружен (пустой лист, сплошной фон) — разрешается объектом с `angle: 0` и `deskewedImage: null` (или исходным изображением, зависит от контракта вызова).

### 4. Алгоритм обработки

#### 4.1. Убедиться в grayscale
Если входное изображение цветное или с альфа-каналом — конвертировать в 8-бит серый через `sharp.grayscale()`. Поскольку ожидается всегда grayscale, этот шаг — предохранитель.

#### 4.2. Выделение краёв (детектор градиента)
- Применить свёртку с ядрами Собеля 3×3 для получения горизонтальной и вертикальной составляющих градиента.
- Вычислить магнитуду `M = sqrt(Gx² + Gy²)` (либо для скорости `|Gx| + |Gy|`).
- Бинаризовать магнитуду с порогом `edgeThreshold`: пиксели с магнитудой ≥ порога становятся белыми (`255`), остальные чёрными (`0`).

Результат: чёрно-белое изображение, где белые пиксели соответствуют резким перепадам яркости (края документа, текст, тени).

#### 4.3. Морфологическая обработка краёв
- Применить несколько итераций дилатации (расширения) с квадратным структурным элементом 3×3. Это соединит разорванные фрагменты краёв документа в сплошную линию.
- Применить столько же итераций эрозии (сужения) для восстановления приблизительной исходной толщины линии.

Результат: бинарное изображение, где контур документа представляет собой практически непрерывную белую рамку, а текстовые артефакты либо исчезли, либо превратились в мелкие шумовые пятна.

#### 4.4. Сбор пикселей контура
- Извлечь все координаты `(x, y)` белых пикселей (значение `255`).
- Для ускорения допустимо взять каждый второй или каждый N-й пиксель (проредить), если точек очень много (например, > 100 000). Это не должно заметно повлиять на точность minAreaRect.

#### 4.5. Итеративный поиск минимального ограничивающего прямоугольника (minAreaRect)
- Вычислить выпуклую оболочку собранных точек (алгоритм Эндрю, монотонная цепочка).
- Двухпроходный итеративный поиск:
  1. **Грубый проход:** перебрать углы от 0° до 180° с шагом 5°. Для каждого угла повернуть точки оболочки на этот угол, найти ограничивающий выровненный по осям прямоугольник, вычислить его площадь. Запомнить угол с минимальной площадью.
  2. **Точный проход:** в диапазоне найденного угла ±5° выполнить проход с шагом 0.1° (или 0.05°). Найти финальный угол и соответствующий прямоугольник (четыре вершины в исходных координатах).
- Вершины прямоугольника упорядочить в порядке: верхний левый, верхний правый, нижний правый, нижний левый (с учётом направления оси Y вниз).
- Вычислить угол поворота как угол вектора от верхнего левого к верхнему правому углу относительно горизонтали, привести к диапазону [-45°, +45°] или как удобно для `sharp.rotate()`. Угол должен быть таким, чтобы после поворота строки текста стали горизонтальны.

#### 4.6. Выпрямление
- Повернуть исходное grayscale-изображение на вычисленный угол с помощью `sharp.rotate(angle, { background: 255 })` (белый фон).
- Обрезать белые поля, оставшиеся после поворота (`sharp.trim(threshold)` с порогом, близким к 255, чтобы убрать только добавленные вращением белые области).
- Добавить отступ `padding` со всех сторон (`sharp.extend({ top, bottom, left, right, background: 255 })`).
- Конвертировать в PNG буфер.

#### 4.7. Определение ориентации
- Использовать ширину и высоту минимального ограничивающего прямоугольника, полученного на этапе 4.5.
- Если ширина > высоты — `"landscape"`, иначе `"portrait"`.

### 5. Требования к реализации
- **Среда:** Node.js 20+, строго без нативных аддонов, кроме `sharp`.
- **Зависимости:** `sharp`.
- **Производительность:** обработка одного изображения А4 при 300 DPI не должна превышать 500 мс на типовом сервере.
- **Тестирование:** unit-тесты на каждом шаге (бинаризация градиента, морфология, minAreaRect), интеграционные тесты на наборе эталонных grayscale-сканов.

### 6. Критерии приёмки
- На 100 тестовых сканах с белым фоном, содержащих одиночные документы с поворотом от -45° до +45°:
  - Угол перекоса определяется с погрешностью ≤ 0.5°.
  - Выровненное изображение не содержит остаточного видимого наклона.
  - Документ не обрезан (в пределах паддинга).
  - Время обработки ≤ 500 мс.
